Detailed Description

A data structure for forwarding tables.

Like Trie2, this implementation stores values at their minimum required depth instead of always at depth 32. It's storage use is better than Trie0 and Trie 1, but is still suboptimal, even for this implementation. It may be extendable such that fewer CIDR-style routing entries are effectively stored.

In contrast to Trie2, this Trie supports unordered CIDR input. On a collision with a space-optimized CIDR address already in the tree (ie an address that is stored higher in the tree to save nodes) it will attempt to propagate the node down the tree to some maximum depth (the length of its original mask). This adds light overhead to insert(). It aims to reduce memory overhead, but requires slightly more memory per TrieNode and thus requires extensive testing.

NOTE: The CIDR address loading order limitations noted for Trie0 and Trie1 do not apply to this class.

Inserts a new (key, data) pair.

Adds a new (key, data) pair to the Trie. If the key already exists, and if the replace flag is set, we will replace the old data with the new one and return the old data. If the replace flag is not set, the old data remains in the trie and the new data is returned. If the key does not exist in trie, the data will be inserted and the return value is 0.

Parameters
keythe key to insert
nBitsbits of the key to consider (useful for CIDR)
datathe TrieData object to add
replacewhether to replace an existing value with the same key
Returns
NULL on success, otherwise the TrieData object we attempted to insert (so reference is not lost)
