1 /*
2  * barrier.cc
3  *
4  * A barrier causes threads to wait until a set of threads has
5  * all "reached" the barrier. The number of threads required is
6  * set when the barrier is initialized, and cannot be changed
7  * except by reinitializing.
8  *
9  * The barrier_init() and barrier_destroy() functions,
10  * respectively, allow you to initialize and destroy the
11  * barrier.
12  *
13  * The barrier_wait() function allows a thread to wait for a
14  * barrier to be completed. One thread (the one that happens to
15  * arrive last) will return from barrier_wait() with the status
16  * -1 on success -- others will return with 0. The special
17  * status makes it easy for the calling code to cause one thread
18  * to do something in a serial region before entering another
19  * parallel section of code.
20  *
21  * min_value_reset initializes the min value to -1 so that the
22  * reduction can be used for the next synchronization
23  */
