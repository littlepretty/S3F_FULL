225  ltime_t Entity::d2t(double d)
226  ltime_t Entity::d2t(double d, int s)
227 
228  Conversion routines for ticks and doubles.
229 
230  S3F's clock is discrete, fixed point binary where
231  the position of the binary point is declared when
232  the simulation is constructed (units in
233  Timeline::__log_ticks_per_sec).
234  Often we need to
235  representing floating point numbers in this discrete
236  format, and vice-versa. Methods t2d and d2t are provided
237  for this translation. The version version assumes that the
238  double is expressed in the same units as the clock ticks.
239 
240  t2d transforms ltime_t -> double
241  d2t transforms double -> ltime_t
242 
243  The second version includes a scale factor s describing the double.
244  s is likewise in log_10(clock ticks per sec).
245 
246  That is, given clock tick variable C and
247  scale factor s, t2d will transform C into a double length
248  floating point representation of C x 10^{-s} virtual seconds.
249  d2t with scale argument s means that the units of the input
250  are 10^{s} per unit second, and will create a ltime_t value
251  at the scale that all clock values share.
252 
253  **************************************************************/
254
